# Transup 待补齐功能点

面向"现代 harness"标准，梳理当前架构还欠缺、需要补齐的能力。按优先级排列。

## 1. 运行时韧性（最高优先级）

现代 harness 的真实杀手不是缺功能，而是 agent loop 在长任务里断档。需要在 `core/agent/engine.ts` 补：

- **防断档续跑**：模型停在一次 tool call、只解释不继续执行时，自动续跑而非结束回合。
- **中断恢复质量**：Ctrl+C / approval 等待后，上下文和"当前工作文件"不丢失。
- **异常自愈**：malformed tool call、未知工具、非法参数的有限次重试与降级。
- **循环保护**：重复 tool 结果 / 重复 assistant 消息的检测与打断，避免空转。

> 实现原则：以能力形式补进引擎，保持文件精简，不堆成巨型单文件。
