## 1. 输入模式系统（bash `!`）

### 1.1 模式定义
`src/types/textInputTypes.ts:265`：`PromptInputMode = 'bash' | 'prompt' | 'orphaned-permission' | 'task-notification'`。

**注意：此版本已没有 `memory`（`#`）输入模式**——`inputModes.ts` 只识别 `!`；`#` 前缀被重新用于 Slack 频道补全（useTypeahead.tsx:42,640-649）。仅存遗留渲染器 `src/components/messages/UserMemoryInputMessage.tsx`。Transup 若要实现 memory 模式，可完全类比 bash 模式实现。

### 1.2 bash 模式状态机（`src/components/PromptInput/inputModes.ts`，全 33 行）
- `getModeFromInput(input)`：`startsWith('!')` → 'bash'。
- `prependModeCharacterToInput`：存历史时把 `!` 加回去。
- **进入**：在光标位置 0 键入 `!`：
  1. `useTextInput.ts:404`：`if (cursor.isAtStart() && isInputModeCharacter(input)) return cursor.insert(text).left()`——插入后光标左移回 0（妙处：让上层能把它识别为"起始处单字符插入"）。
  2. `PromptInput.tsx` onChange（:869-887）：检测"单字符插入 + cursorOffset===0 + mode!=='prompt'" → 只调 `onModeChange(mode)` 并 **return，不把 `!` 写入 input**（`!` 变成模式而非文本）。多字符插入到空输入（tab 补全整条 bash 历史、粘贴 `!cmd`）则剥掉 `!` 前缀后写入。
- **退出**：PromptInput.tsx ~1905：`cursorOffset === 0 && (escape || backspace || delete || ctrl+u)` → `onModeChange('prompt')`。
- **视觉**：
  - 边框色 `getBorderColor()`（PromptInput.tsx 尾部）：bash → theme `bashBorder`（亮粉 `rgb(255,0,135)`，深色主题，theme.ts:117；ANSI 回退 magenta）。默认 `promptBorder` 灰 `rgb(153,153,153)`。
  - 提示符：`PromptInputModeIndicator.tsx:82`——bash 模式渲染 `<Text color="bashBorder">! </Text>`；普通模式渲染 `figures.pointer`（`❯ `），isLoading 时 dim；teammate 视图用 agent 颜色。
  - footer 左侧：`PromptInputFooterLeftSide.tsx:317` `mode==='bash'` → `<Text color="bashBorder">! for bash mode</Text>`。
- **历史联动**：bash 模式下按 ↑ 只翻 bash 历史（useArrowKeyHistory.tsx:132 `initialModeFilterRef`）。

## 4. 核心编辑键位（useTextInput.ts）

### 4.1 emacs 键位表（:224-245）
- Ctrl：`a`行首 `b`左 `e`行尾 `f`右 `h`退格(先试 deleteTokenBefore) `k`kill到行尾 `u`kill到行首 `w`kill前一词 `y`yank `n`/`p`下/上(或历史) `d`/`c`（见 §5）。
- Meta：`b`/`f` 词移动 `d`删后词 `y`yank-pop。
- **kill ring**（Cursor.ts:16-115）：模块级，容量 10；连续 kill 累积（`append`/`prepend` 方向），非 kill 键 `resetKillAccumulation`；`Ctrl+Y` yank 后 `Alt+Y` 轮换（yank-pop 链，任何其他键 `resetYankState` 打断）——完整 readline 语义。
- Home/End、`\x1b[H`/`\x1b[1~`/`\x1b[F`/`\x1b[4~` 原始序列（:385-389）；PageUp/PageDown 映射行首/行尾但全屏模式下 no-op（交给滚动系统）；滚轮事件 no-op 防止 SGR 序列被当文本插入（:360-365）。
- Meta+Backspace = killWordBefore，Meta+Delete = killToLineEnd（:336-341）。

### 4.2 Enter / 换行（handleEnter，:247-267）
1. `\` + Enter → 删掉反斜杠插入 `\n`（并 `markBackslashReturnUsed()` 用于提示文案进化：`backslash (\) + return (⏎) for newline` → `\⏎ for newline`，见 `PromptInput/utils.ts` getNewlineInstructions）。
2. Meta+Enter 或 Shift+Enter → 插入 `\n`。
3. **Apple Terminal hack**：不支持自定义 Shift+Enter 绑定，用原生 macOS API 检测 Shift 是否按下：`env.terminal === 'Apple_Terminal' && isModifierPressed('shift')`；hook 开头 `prewarmModifiers()` 预热（:98-101）。
4. 否则 `onSubmit(value)`。
- **SSH 合帧 Enter**（:391-499 两处）：慢链路上 "o"+Enter 会合成一个 chunk `"o\r"`——默认分支剥掉尾部 `\r`；onInput 末尾检测"多字符输入 + 单个尾随 \r + 前一字符非 `\\`" → 触发提交；单独 `\r` 是 Alt+Enter 泄漏 → 转 `\n`；内嵌 `\r` 是无 bracketed-paste 终端的多行粘贴 → 转 `\n`；`\\\r` 兼容旧版 VS Code /terminal-setup 写坏的 Shift+Enter 绑定（issue #31316）。
5. Tab 制表符全局替换为 4 空格（onChange 内 `replaceAll('\t','    ')`）。

### 4.3 SSH/tmux 退格兼容（:442-465）
非退格键事件却含 `\x7f`（Issue #1853）：统计 DEL 数量，同步执行 N 次 `deleteTokenBefore() ?? backspace()`，一次性 setState。

### 4.4 上下键 = 光标 or 历史（:269-316）
`upOrHistoryUp`：先试视觉折行 `cursor.up()` → 再试逻辑行 `upLogicalLine()` → 都动不了才 `onHistoryUp()`。有补全下拉（suggestions>0）或 footer pill 选中时 `disableCursorMovementForUpDownKeys` 直接走历史/导航回调。

## 5. 双击退出 / 清空 / Esc 状态机

### 5.1 useDoublePress（`src/hooks/useDoublePress.ts`）
通用原语：`DOUBLE_PRESS_TIMEOUT_MS = 800`；`(setPending, onDoublePress, onFirstPress?)`；首按 → setPending(true) + 800ms 定时回落。四个实例：

| 实例 | 首按 | 二按 |
|---|---|---|
| Ctrl+C（useTextInput.ts:108-120） | footer 显示 `Press Ctrl-C again to exit`；若有输入，**首按即清空输入+重置历史** | `onExit()` |
| Ctrl+D 空输入（:155-168） | 同样 exit 提示（key='Ctrl-D'） | `onExit()`；**非空输入时 Ctrl+D = 向前删除一个字符（iPython 风格，:170-178）** |
| Esc 清空（:126-153） | 有输入时通知 `Esc again to clear`（1s） | 清空输入，**先 addToHistory 保存**，重置历史游标 |
| Esc 空输入（PromptInput.tsx:1254） | 无 | 打开 message selector（rewind 回溯对话） |

exitMessage 状态通过 `onExitMessage(show, key)` 冒泡到 footer（PromptInputFooterLeftSide.tsx:147-157）。

### 5.2 Esc 全局优先级链（PromptInput.tsx:1915-1955 + useCancelRequest.ts）
自上而下：模态对话框打开 → 全跳过；abort speculation；dismiss side question；关闭 help 菜单；footer pill 选中 → 交给 Footer 上下文 `footer:clearSelection`；有可编辑的排队消息 → **Esc 把队列弹回输入框编辑**；`messages>0 && input空 && !isLoading` → 双击 Esc 开 message selector。加载中：`chat:cancel` → `useCancelRequest.ts` handleCancel：优先 abort 当前任务（清 permission 队列），否则弹出队列消息。补全下拉打开时 Esc 被 Autocomplete 上下文抢走（dismiss），且 autocomplete 注册为 overlay 使 CancelRequestHandler 让位（useTypeahead.tsx:1265-1272）。文本层双击 Esc 在 suggestions>0 时禁用（`disableEscapeDoublePress`）。

## 6. 粘贴系统

### 6.1 检测与聚合（`src/hooks/usePasteHandler.ts`，285 行）
- 判定为粘贴（:253-258）：bracketed paste 标记 `event.keypress.isPasted`（由 keypress 解析器置位——此前 stdin 双监听器竞争导致丢字符，:208-212 注释）‖ 单帧输入 > `PASTE_THRESHOLD`（=800，`src/utils/imagePaste.ts:30`）‖ 上一个 chunk 还在 pending ‖ 内容像图片文件路径。
- chunk 聚合：每个 chunk 重置 100ms 定时器（`PASTE_COMPLETION_TIMEOUT_MS`），超时后 join 处理；**`pastePendingRef` 是 React state 的同步镜像**（:48-53 注释）：粘贴+按键同帧到达时，第二次调用读 state 会是 stale null，若那个键是 Enter 会提交旧输入丢失粘贴——用 ref 同步标记规避。
- join 后清洗：去掉被焦点事件切断的 `[I`/`[O` 残片（:113-116）。
- **拖拽图片路径**：按 `/ (?=\/|[A-Za-z]:\\)/` + 换行切分（空格内的路径已被转义，:120-130），`isImageFilePath` 过滤，全部读为图片；非图部分继续按文本粘贴；macOS 临时截图路径（`/TemporaryItems/…screencaptureui…`）文件已消失时回退查剪贴板（:132-168）。
- **空粘贴 = 剪贴板图片**（macOS，Cmd+V 粘图时终端发出空的 bracketed paste）：`checkClipboardForImage`（50ms debounce，:89-92）。
- isPasting → footer `Pasting text…`；BaseTextInput.tsx:59-63 粘贴期间吞掉 Enter，防止把多行粘贴中的回车当提交。任何 >10 字符输入也强制关掉 isPasting（stdin 缓冲可能把关闭转义序列切断，:271-277）。

### 6.2 大段文本折叠（PromptInput.tsx onTextPaste，:1201-1240）
- 清洗：stripAnsi、`\r`→`\n`、tab→4 空格；空输入时粘 `!cmd` 自动进 bash 模式。
- 折叠条件：`text.length > 800 || 行数 > maxLines`，其中 `maxLines = min(rows-10, 2)`（注释：布局过高会让 Ink 全屏重绘）。
- 折叠为 `PastedContent{id, type:'text', content}` 存入 `pastedContents`，输入框只插入占位符 `[Pasted text #N +M lines]`（`src/history.ts:51-56`；M=换行数，即 3 行显示 +2）。
- 提交/入历史时占位符与真实内容通过 `parseReferences`（history.ts:62，正则 `\[(Pasted text|Image|\.\.\.Truncated text) #(\d+)( \+\d+ lines)?\]`）还原。
- ID 计数器 `nextPasteIdRef`（图片与文本共用），初值扫描历史 messages 取 max+1（`getInitialPasteId`）以支持 --resume 不冲突；懒初始化避免每次渲染重扫（:361-369 注释）。
- **超长输入截断**（非粘贴路径，如外部注入）：`useMaybeTruncateInput.ts` + `inputPaste.ts`：>10000 字符 → 首尾各留 500，中段折为 `[...Truncated text #N +M lines...]`。

### 6.3 图片粘贴
- 快捷键 `chat:imagePaste` = Ctrl+V（Windows 是 Alt+V，因 Ctrl+V 是系统粘贴，defaultBindings.ts:12-15）→ `getImageFromClipboard`（imagePaste.ts）：macOS 优先原生 NSPasteboard（约 5ms，osascript 要 ~1.5s），回退 `osascript «class PNGf»`；Linux `xclip`/`wl-paste`；Windows PowerShell `Get-Clipboard -Format Image`。无图时通知 `No image found in clipboard...`（SSH 环境改为 "You're SSH'd; try scp?"）。
- `onImagePaste`（PromptInput.tsx:1151-1183）：强制切回 prompt 模式；`cacheImagePath` 立即缓存路径（渲染可点）+ `storeImage` 后台落盘；插入 `[Image #N]` chip。
- **chip 交互细节**：
  - `pendingSpaceAfterPillRef`（:370-373, 1241-1246）：粘图后武装标记，**下一个非空格可打印字符自动补一个前导空格**；其他任何键（方向、Esc、退格、再粘贴）拆除标记。多图连粘时上一个 pill 的懒空格先落地（:1177-1182）。判定函数 `isNonSpacePrintable`（PromptInput/utils.ts）。
  - 光标落在 chip.start 时整个 chip 反色显示 = "选中"态（highlight `inverse:true`，priority 8，:606-615），同时 `showCursor=false` 隐藏字符光标；光标落进 chip 内部（点击/上下移动）自动吸附到最近边界（:594-600）。
  - 孤儿清理 effect（:1189-1200）：任何编辑删掉 `[Image #N]` 文本，对应 pastedContents 自动删除。
- 焦点回归提示（`useClipboardImageHint.ts`）：终端重获焦点 + 剪贴板有图 → 通知 `Image in clipboard · ctrl+v to paste`（1s debounce、30s 冷却）。

## 7. 历史记录

### 7.1 上下键（`src/hooks/useArrowKeyHistory.tsx`）
- 存储：全局历史文件，`MAX_HISTORY_ITEMS=100`（history.ts:19），entry 含 `display`（带 `!` 前缀）+ `pastedContents`。
- 触发条件（PromptInput.tsx handleHistoryUp/Down，:924-967）：suggestions ≤1；光标在首/末行（多行时先移光标）；有可编辑队列消息时 ↑ 先弹队列。
- **性能设计**：按 `HISTORY_CHUNK_SIZE=10` 分块懒加载；`pendingLoad` 模块级 Promise 合并并发读盘（快速连按共享一次 IO，:15-62）；`historyIndexRef` 同步自增规避 setState 异步下的 stale index（:86-88）；越界时回滚 ref。
- 草稿保存：首次 ↑ 时把当前输入（含 mode、pastedContents）存为 `lastShownHistoryEntry`，↓ 回到 0 时恢复。
- 历史条目上屏后**光标置于开头**（`cursorToStart=true`）。
- 导航 ≥2 条后每 session 提示一次 `ctrl+r to search history`。
- ↓ 到底（index 0）返回 true → 进入 footer pill 导航（:957-966）。

### 7.2 Ctrl+R 增量搜索（`src/hooks/useHistorySearch.ts`）
- `history:search` 绑定 Ctrl+R（Global）；进入时快照 original input/cursor/mode/pastedContents。
- 搜索：`makeHistoryReader()` 反向异步生成器逐条读文件；`display.lastIndexOf(query)` 子串匹配；`seenPrompts` 去重；**匹配即实时替换主输入框内容**（bash 模式也切），光标定位到匹配处；查询变更 abort 旧搜索（AbortController，:286-294）；生成器显式 `.return()` 关闭文件句柄防 fd 泄漏（:51-58 注释）。
- 键位（HistorySearch 上下文，defaultBindings.ts:171-180）：`ctrl+r`=下一个匹配（resume 生成器）、`escape`/`tab`=接受留在输入框、`enter`=直接提交执行、`ctrl+c`=取消恢复原输入；query 为空时 backspace 取消（:262-268）。
- 视觉：footer 左侧换成 mini 搜索框（`HistorySearchInput.tsx`）：dim 标签 `search prompts:`，匹配失败变 `no matching prompt:`（保留上一个匹配）；主输入框内匹配片段高亮 `warning` 黄色（priority 20，PromptInput.tsx:617-624）；搜索时主输入 `focus=false, showCursor=false`。
- **HISTORY_PICKER 特性开关**下 Ctrl+R 改为模态 `HistorySearchDialog.tsx`：FuzzyPicker，精确子串优先 + 字符子序列 fuzzy 兜底（isSubsequence）；≥100 列时右侧 6 行预览面板，超出显示 `… +N more lines`；年龄列宽 8。

## 8. 补全系统（useTypeahead.tsx，1384 行）

### 8.1 斜杠命令
- 条件：prompt 模式、`/` 开头、光标不在 `/` 上、命令后没有实参（`hasCommandWithArguments` :343-348 处理"回头改命令名"场景）。
- 匹配：Fuse.js（commandSuggestions.ts:53-77），threshold 0.3、location 0（偏好前缀），权重 commandName 3 / 命令名分段（`:_-` 切词）2 / alias 2 / 描述词 0.5；**Fuse 索引按 commands 数组身份缓存**，不逐键重建（:22-28）。
- 下拉打开即默认选中第 0 项；`maxColumnWidth` 用**全部命令**的最长名计算（+6），防止过滤时布局抖动（useTypeahead.tsx:378-386）。
- Tab=补全不执行（`applyCommandSuggestion(s, false)`），Enter=补全并执行（`true`，并以 `isSubmittingSlashCommand=true` 绕过"有下拉禁止提交"守卫）；↑/↓/Ctrl+P/Ctrl+N 循环选择；Esc dismiss 且 `dismissedForInputRef` 记住当前输入防止立刻重触发（:1233-1239）。
- **句中斜杠命令 ghost text**：`help me /com` → `findMidInputSlashCommand`（前面必须是空白，:114-142）→ 灰色内联补全后缀；**用 useMemo 同步计算而非 useEffect，消除一帧闪烁**（:400-414 注释）；Tab 展开为 `/command `。ghost 首字符渲染进反色光标内、余下 dim（Cursor.ts render :270-290）。
- 参数提示：恰好一个尾随空格时显示静态 `argumentHint` 或按 `argNames` 生成渐进提示（:729-774）。

### 8.2 `@` 文件/资源引用
- token 正则 Unicode 感知（`\p{L}\p{N}\p{M}` 支持 CJK/NFD 重音，:33-41）；支持 `@"含 空格的文件"` 引号形式；`@` 快速路径用 `lastIndexOf('@')` 避免 `$` 锚点全扫（:289-306）。
- 路径型 token（`@./ @~ @/`）走目录遍历补全 `getPathCompletions`；否则统一 fuzzy（文件 `+`、MCP 资源 `◇`、agent `*` 图标，PromptInputFooterSuggestions.tsx:24-29）。
- **性能**：挂载即后台预热文件索引（带 ~4ms event-loop yield 的分块构建；用户先打字给部分结果，索引完成后自动重发最后一次搜索升级为全量，:485-505）；**50ms debounce，注释明确"卡在 macOS 按键重复 ~33ms 之上"，长按删除合并为一次搜索**（:507-511）；`latestSearchTokenRef` 丢弃过期异步结果。
- 接受：Tab 先试公共前缀补全（不关下拉），否则整项替换 `@path `（目录补 `/` 并继续拉取下一层，文件补空格关闭）；含空格文件自动加引号；bash 模式下补全**不带 @**（formatReplacementValue :148-173）。
- `@name` teammate/agent DM 建议优先于文件（:593-637）；`#channel` → Slack 频道（150ms debounce）。

### 8.3 bash 模式补全
- 输入时：shell 历史 ghost text（`getShellHistoryCompletion`），Tab 接受整条历史。
- 无 ghost 时 Tab：`getShellCompletions`（命令/变量/文件），**唯一结果直接应用**，多结果开下拉（:1093-1109）。

### 8.4 下拉渲染（`PromptInputFooterSuggestions.tsx`）
- 下拉直接**替换 footer 区域**（PromptInputFooter.tsx 早退分支，paddingX 2）；全屏模式 portal 到 overlay（最多 5 项）。
- 非全屏最多 `min(6, rows-3)` 行，滚动窗口让选中项居中；选中项 `suggestion` 蓝（`rgb(87,105,247)`），其余 dim；文件路径中段截断 `truncatePathMiddle`，描述压缩空白后尾部截断；命令列固定宽 + `[tag]` + 描述。

### 8.5 提交守卫
`onSubmit`（PromptInput.tsx:984-1105）：下拉可见时 Enter 不提交（目录型例外，Tab 用于补全）；footer pill 选中不提交；空输入无图不提交；空输入/等于建议文本时接受 AI prompt suggestion（speculation 活跃时走秒回路径）；`@name msg` 走 DM；teammate 视图路由到 `onAgentSubmit`。

## 9. 快捷键体系（src/keybindings/）

### 9.1 架构
- **上下文栈**：Global / Chat / Autocomplete / Footer / HistorySearch / Help / Confirmation / Select / Transcript / Scroll / MessageSelector / Task / …（defaultBindings.ts 全表）。组件用 `useKeybinding(s)(action, handler, {context, isActive})` 注册；`useRegisterKeybindingContext` 声明上下文活跃。
- 解析（resolver.ts）：遍历所有 binding，**last-wins**（用户覆盖）；`action: null` = 显式解绑（`unbound` 吞键）。chord 支持（`ctrl+x ctrl+e`），`chord_started/cancelled` 状态。
- 兼容细节：alt 与 meta 合并匹配（旧终端无法区分，match.ts:50-80）；`super`（Cmd/Win）独立、仅 kitty 键盘协议终端会送达；**Ink 的 escape 会带 key.meta=true 的怪癖被显式剥掉**（resolver.ts buildKeystroke 注释）。
- 用户自定义：`~/.claude/keybindings.json`（loadUserBindings.ts:116），带文件 watcher 热重载；`reservedShortcuts.ts`：ctrl+c / ctrl+d / ctrl+m(=Enter) **不可重绑**。
- 巧妙点：`chat:submit` **不经 useKeybindings 注册**而是直接进 handler registry（PromptInput.tsx:1637-1653 注释）——否则 Enter 会被 stopImmediatePropagation，补全就看不到 Enter；默认 Enter 由 TextInput onSubmit 与 useTypeahead 处理，registry 里的仅供 chord（如 `ctrl+e s`）调用。

### 9.2 默认键位总表（defaultBindings.ts:32-97 + 硬编码）
- Global：`ctrl+c` 中断（双击退出）、`ctrl+d` 退出、`ctrl+l` 重绘、`ctrl+t` todos、`ctrl+o` transcript/verbose 切换、`ctrl+r` 历史搜索。
- Chat：`escape` 取消、`shift+tab`（win 无 VT 时 `meta+m`）模式循环、`meta+p` 模型选择、`meta+o` fast mode、`meta+t` thinking 开关、`enter` 提交、`up/down` 历史、`ctrl+_` 与 `ctrl+shift+-` 撤销（**两个绑定：legacy 终端发 \x1f，kitty 发物理键**，:77-81 注释）、`ctrl+x ctrl+e` 与 `ctrl+g` 外部编辑器、`ctrl+s` stash、`ctrl+v`（win `alt+v`）图片粘贴、`shift+up` message actions、`ctrl+x ctrl+k` 杀 agent、space 语音（按住说话）。
- Autocomplete：`tab` accept、`escape` dismiss、`up/down` 选择。Task：`ctrl+b` 后台化。Footer：方向键 + enter + escape。
- `?`：不是键绑定——onChange 里 `value === '?'` 时 toggle 帮助菜单（PromptInput.tsx:854-859），即输入框里只打一个 `?` 就开帮助；vim NORMAL 下也特判（useVimInput.ts:289-295）。帮助菜单内容（PromptInputHelpMenu.tsx）：`! for bash mode`、`@ for file paths`、`/ for commands`、`double tap esc to clear input`、换行说明、stash/undo/model/fast/verbose/suspend 等。
- **macOS Option 误配检测**（PromptInput.tsx:1873-1888）：Option+字母产生特殊字符（如 π）时弹提示 "To enable ⌥…, set **Option as Meta** in <终端名> preferences (⌘,)" 或 "run /terminal-setup"，且**不拦截字符**，让用户看到问题。

### 9.3 撤销 / stash / 外部编辑器
- 撤销（`useInputBuffer.ts`）：环形快照 50 条、**1000ms debounce 合并连续击键**；快照含 text+cursor+pastedContents；`ctrl+_` 恢复。
- stash（`handleStash`，:1357-1383）：非空 → 存 {text, cursorOffset, pastedContents} 并清空；空 + 有 stash → 弹回。发现机制：输入从峰值 ≥20 渐进删到 ≤5（排除一步清空）且未用过 stash → 提示 `Tip: ctrl+s to stash`（:783-830）。
- 外部编辑器（`handleExternalEditor`，:1321-1354）：$EDITOR 编辑，**展开折叠的粘贴引用**，期间输入框显示占位横幅。

## 10. Footer pill 导航与队列

- 输入区下方的 pill（tasks/tmux/teams/bridge/companion）构成导航序列（footerItems，:450-460）；光标在末行再按 ↓ 进入第一个 pill；↑/↓/←/→ 在 pill 与 agent 行间移动、Enter 打开、Esc 清除选择；**type-to-exit**：pill 选中时敲任何可打印字符回焦输入框并键入该字符（:1893-1901）。pill 选中时 TextInput `focus=false`，所以 ↑/↓ 由 Footer 上下文键位接管（:1739-1741 注释）。
- 排队消息（`PromptInputQueuedCommands.tsx`）：加载中提交的消息渲染在输入框上方；↑ 或 Esc 弹回可编辑（popAllEditable）；task-notification 最多显示 3 条，溢出合成 `+N more tasks completed`。

## 11. Vim 模式（src/vim/ + useVimInput.ts）

- 开关：全局 config `editorMode === 'vim'`（PromptInput/utils.ts isVimModeEnabled）；`VimTextInput` 替换 `TextInput`，vimMode 状态提升到 PromptInput props 以便跨重挂载保留。
- 状态机（`vim/types.ts:1-90`，注释里有完整状态图）：`VimState = INSERT{insertedText} | NORMAL{command}`；`CommandState = idle | count | operator | operatorCount | operatorFind | operatorTextObj | find | g | operatorG | replace | indent`。`transitions.ts` 是纯转移函数，`operators.ts` 执行 d/c/y、x、r、~、J、p/P、>/<、o/O、D/C、yy 等；`PersistentState` 存寄存器（含 linewise 标志）、lastFind（`;`/`,` 重复）、lastChange（`.` dot-repeat 通过重放 RecordedChange 实现，useVimInput.ts:112-175）。
- 关键路由（useVimInput.ts handleVimInput，:180-296）：Ctrl 组合键**直接透传给 emacs 层**（vim 用户仍可用 ctrl+a/e）；Esc INSERT→NORMAL（注释声明**故意不进 keybinding 系统，不可配置**），退出时光标左移 1（vim 语义）；NORMAL 下 Esc 取消 pending 命令；Enter 任何模式都透传（NORMAL 下也能提交）；NORMAL idle 下方向键透传（保留历史回退）；其余方向键映射 hjkl；Backspace/Delete 仅在"期待 motion"的状态映射 h/x（避免 `r<Backspace>` 变成"替换为 h"，:255-263 注释）；`u` 调用外层 undo buffer。
- 视觉：INSERT 时 footer `-- INSERT --`（dim）；inputFilter（懒空格）所有模式跑但只在 INSERT 应用结果（:43-47 注释，防止 Esc 往返后过期武装）。

## 12. 渲染与高亮细节

- **高亮合成**（PromptInput.tsx combinedHighlights，:601-741），按 priority：历史搜索匹配 20（warning 黄）> `btw` 15（黄）> ultrathink/ultraplan/ultrareview/buddy **逐字符彩虹** 10（`getRainbowColor(i)`，附 shimmerColor）> 图片 chip 反色 8 > 斜杠命令/token budget/slack/成员 @mention 5（蓝 `suggestion` 或 agent 色）> 语音临时文本 dim 1。
- 光标所在的高亮被过滤掉，保证光标反色可见（BaseTextInput.tsx:93）；视口窗口化时高亮坐标做 offset 重映射（:98-102）。
- shimmer（`ShimmeredInput.tsx`）：`useAnimationFrame(50ms)`，扫光索引 = `sweepStart + (t/50) % cycleLength`，sweep 范围 = 所有 shimmer 高亮的 [lo-10, hi+20)。ultrathink 键入时还弹通知 `Effort set to high for this turn`（:748-758）。
- 掩码输入（OAuth token 等，Cursor.ts render :222-238）：**只显示最后一行末 6 个字符**，前面全 mask；折行的前几行全 mask（修复过窄终端泄漏 token 开头的 bug）。
- 语音波形光标（TextInput.tsx:15-91）：录音时光标变成音量柱（9 级块字符 ` ▁▂▃▄▅▆▇█`），EMA 平滑 0.7、增益 1.8、静音阈值 0.15 显示灰色，色相 90°/s 旋转。
- 点击定位光标（全屏 POC，handleInputClick）：`MeasuredText.getOffsetFromPosition({line: localRow+viewportStart, column: localCol})` 处理宽字符/折行/越界钳制。
- 外部文本注入（STT）：`lastInternalInputRef` 区分内部编辑与外部替换，外部变更自动把光标移到末尾；`insertTextRef` 暴露 `insert()`（智能补空格）与 `setInputWithCursor()`（:252-285）。

## 13. Transup 实现优先级（按依赖序）

1. 不可变 Cursor + MeasuredText（grapheme/宽字符/折行/视口）——一切编辑的地基
2. useTextInput 的 emacs 键位 + 双击 Ctrl-C/D/Esc 原语（800ms）
3. usePasteHandler（bracketed paste + 100ms 聚合 + pendingRef 同步镜像）与 `[Pasted text #N]` 折叠
4. 上下文化 keybinding 解析器（last-wins、alt/meta 合并、保留键）
5. 模式系统（`!` 前缀的"单字符插入即模式"技巧 + 边框/提示符换色）
6. 历史（分块懒加载 + 草稿保存 + Ctrl-R 反向增量搜索实时替换）
7. typeahead（同步 ghost text、50ms debounce、stale-token 丢弃、公共前缀 Tab）
8. 视觉层（round 无侧边框、⏸/⏵⏵ 模式指示、? 帮助、placeholder 首字符反色光标）
