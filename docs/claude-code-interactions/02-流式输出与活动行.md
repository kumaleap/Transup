# 02 流式输出与活动行（Spinner）

> 源码引用相对 `claude-code-analysis` 仓库根目录。

## 1. 主活动行（Spinner 行）

### 1.1 整体结构

组件链：`REPL.tsx` → `SpinnerWithVerb`（`src/components/Spinner.tsx`）→ `SpinnerAnimationRow`（`src/components/Spinner/SpinnerAnimationRow.tsx`）。

```
✻ Wibbling…  (2s · ↓ 1.2k tokens · thinking…)
│  │          └── 状态括号：Byline 组件用 " · " 连接各段，整体 dimColor
│  └── 动词 + "…"（U+2026），主题色 claude 橙，带 glimmer 扫光
└── SpinnerGlyph 动画符号，宽度固定 2 列
```

行容器：`<Box flexDirection="row" flexWrap="wrap" marginTop={1} width="100%">`。

### 1.2 Spinner 符号动画（`Spinner/utils.ts`、`SpinnerGlyph.tsx`）

- 帧字符按平台区分（`getDefaultCharacters()`）：
  - macOS（darwin）：`['·', '✢', '✳', '✶', '✻', '✽']`
  - 其他平台：`['·', '✢', '*', '✶', '✻', '✽']`（`✳`→`*`）
  - Ghostty 终端（`TERM=xterm-ghostty`）：`['·', '✢', '✳', '✶', '✻', '*']`（`✽` 渲染偏移）
- 帧序列 = 正序 + 逆序拼接：`[...CHARS, ...CHARS.reverse()]`（12 帧，"呼吸"往返效果）
- **动画时钟：`useAnimationFrame(50)`**（50ms tick，全局共享时钟，终端失焦自动降速，元素离屏暂停）；**帧切换间隔 120ms**：`frame = Math.floor(time / 120)`，`char = FRAMES[frame % 12]`
- 颜色：主题键 `claude` = `rgb(215,119,87)`（ANSI 主题回退 `redBright`；色盲主题 `rgb(255,153,51)`）
- **减少动画模式（prefersReducedMotion）**：符号换静态 `●`，2 秒周期闪烁（1s 正常 / 1s dim），`useAnimationFrame(null)` 关闭 50ms 时钟

### 1.3 停滞变红（`useStalledAnimation.ts`）

- 追踪 `responseLengthRef`（累计流式字符数）。**3000ms 无新 token 且无运行中工具** → `isStalled=true`
- 强度 `intensity = min((timeSince-3000)/2000, 1)`——**2 秒内渐变到全红**
- 每 50ms tick 按 `current += diff * 0.1` 平滑插值；红色 `ERROR_RED = rgb(171,43,63)`，`interpolateColor(claude橙, 红, intensity)` 同时作用于符号和文字
- 有活动工具（`hasActiveTools`）或 leader 空闲时重置计时；变红时 glimmer 扫光关闭（`glimmerIndex=-100`）

### 1.4 动词文案（`src/constants/spinnerVerbs.ts`）

- 内置 **188 个动名词**表 `SPINNER_VERBS`：'Accomplishing'、'Actioning'、…、'Thinking'、'Vibing'、'Wibbling'、'Zigzagging'（含 'Clauding'、'Flambéing'、'Moonwalking' 等彩蛋词）
- 用户可通过 settings `spinnerVerbs` 配置：`mode:'replace'` 替换全表，否则追加（`getSpinnerVerbs()`）
- **选择时机：每个 turn 挂载时随机取一次**（`useState(() => sample(getSpinnerVerbs()))`），turn 内不轮换
- 优先级：`overrideMessage ?? currentTodo.activeForm ?? currentTodo.subject ?? randomVerb`——有进行中 TODO 时用任务的 activeForm（如 "Fixing the bug"）替代随机动词；最终 `message = verb + '…'`

### 1.5 动词上的 glimmer 扫光（`GlimmerMessage.tsx`）

- 一个 3 字符宽的高亮带（`glimmerIndex ± 1`）扫过动词文本，高亮色 `claudeShimmer` = `rgb(245,149,117)`
- 速度：`glimmerSpeed = mode==='requesting' ? 50 : 200`（ms/格）；周期 `cycleLength = 文本宽 + 20`
- **方向**：requesting（发请求中）从左→右：`cyclePos % cycleLength - 10`；其余模式从右→左：`width + 10 - cyclePos % cycleLength`
- `tool-use` 模式额外有整体闪烁：`flashOpacity = (sin(time/1000·π)+1)/2`（2 秒正弦周期），整个动词在 claude↔claudeShimmer 间插值

### 1.6 spinner 模式（SpinnerMode）与流事件映射（`src/utils/messages.ts:2984-3094`）

| 事件 | 模式 |
|---|---|
| `stream_request_start` | `'requesting'` |
| `content_block_start: thinking/redacted_thinking` | `'thinking'` |
| `content_block_start: text` | `'responding'` |
| `content_block_start: tool_use`（及 server_tool_use 等） | `'tool-input'` |
| `message_stop` | `'tool-use'` |
| `message_delta` / 其他 | `'responding'` |

### 1.7 状态括号内各段（`SpinnerAnimationRow.tsx`）

顺序：`spinnerSuffix` → 耗时 → token 计数 → thinking 状态，用 `<Byline>`（`design-system/Byline.tsx`，" · " 中点连接）包裹，外面加 `(` `)`，全部 dim。

- **耗时**：`formatDuration(elapsedMs)`，由 `loadingStartTimeRef/totalPausedMsRef/pauseStartTimeRef` 每帧算 wall-clock（权限弹窗等暂停期扣除）
- **显示门槛**：`wantsTimerAndTokens = verbose || hasRunningTeammates || elapsed > 30_000`——**耗时和 token 数在 30 秒后才出现**（`SHOW_TOKENS_AFTER_MS = 30_000`），除非 verbose
- **token 计数**：`tokens = Math.round(responseLength字符数 / 4)`，`formatNumber` 格式化；前缀符号由 `SpinnerModeGlyph` 决定——`requesting` 时 `↑`，thinking/responding/tool-input/tool-use 时 `↓`；文案 `"↓ 1.2k tokens"`
- **token 数字平滑滚动动画**（50ms 时钟驱动）：gap<70 每帧 +3；gap<200 每帧 `max(8, ceil(gap*0.15))`；否则每帧 +50；reducedMotion 直接跳变
- **thinking 状态**：
  - 思考中显示 `thinking`（可带 effort 后缀，空间不足缩为裸 `thinking`）；文字带"呼吸"发光：3000ms 延迟后 2 秒正弦周期在 `rgb(153,153,153)` ↔ `rgb(185,185,185)` 间插值
  - 结束后显示 `thought for Ns`（`max(1, round(ms/1000))`），**每个状态最少展示 2 秒防跳变**（Spinner.tsx:125-159），持续 2 秒后消失
  - 只有 thinking 一项时渲染成 `(thinking)`
- **逐段宽度裁剪**：`availableSpace = columns - 动词宽 - 2 - 5`，按 thinking → timer → tokens 顺序逐个判断是否放得下

### 1.8 "esc to interrupt" 提示

- **不在 spinner 行**，在输入框下方 footer：`PromptInputFooterLeftSide.tsx` `getSpinnerHintParts()`，isLoading 时渲染 `<KeyboardShortcutHint shortcut="esc" action="interrupt" />` → 文案 **`esc to interrupt`**（dim），后接 ` · ctrl+t to show tasks` 等
- `KeyboardShortcutHint` 格式统一为 `{shortcut} to {action}`，可选括号
- 例外：查看 teammate 转录时 spinner 行尾直接渲染 `(esc to interrupt {teammate名})`，名字用 teammate 颜色（SpinnerAnimationRow.tsx:216）

### 1.9 spinner 下方附加行

- Tip 行：`Tip: {文案}`（dim）；30 秒后且从未用过 /btw → "Use /btw to ask a quick side question…"；**30 分钟**（1_800_000ms）后 → "Use /clear to start fresh…"（Spinner.tsx:256-259）；`spinnerTipsEnabled` 可关
- 有下一个 pending 任务时替代 tip：`Next: {task.subject}`
- Idle 态（leader 完成、teammates 还在跑）：`✻ Idle · teammates running`（dim，`TEARDROP_ASTERISK = '✻'`）
- Brief 模式有独立简化 spinner（单行 shimmer + `N in background` 右对齐 + 断连时红色 `Reconnecting...`）

## 2. 流式文本渲染

### 2.1 数据流（REPL.tsx:1458-1473, messages.ts:3048-3053）

- `content_block_delta: text_delta` 到达 → `setStreamingText(t => t + delta)` **每个 delta 直接 setState，无手动节流**——注释明确："set state directly per delta (**Ink's 16ms render throttle** batches rapid updates)"。节流在自研 Ink 渲染层（16ms ≈ 60fps）
- 同时 `onUpdateLength(delta)` 累加 `responseLengthRef`（驱动 token 计数和停滞检测）；`thinking_delta` 也计入长度但 `signature_delta` 排除
- **按行上屏，不是打字机逐字**：`visibleStreamingText = streamingText.substring(0, lastIndexOf('\n')+1)`——未完成的当前行被隐藏，文本一行一行出现（REPL.tsx:1469-1473）
- `prefersReducedMotion` 或终端有 cursor-up yank bug 时完全禁用流式预览（只在消息完成后整块出现）
- 完整 assistant 消息到达时先 `onStreamingText(() => null)` 再 `onMessage(message)`，同一渲染批次内原子切换，无闪烁/重复（messages.ts:2976-2980）；用户 esc 中断时若 `streamingText` 非空，把已流出的部分固化成 assistant 消息（REPL.tsx:2123-2127）

### 2.2 渲染（Messages.tsx:703-712, Markdown.tsx `StreamingMarkdown`）

- 流式文本渲染在消息列表**动态区尾部**：`⏺`（`BLACK_CIRCLE`，darwin 为 `⏺` 否则 `●`，宽 2 列）+ `<StreamingMarkdown>`
- `StreamingMarkdown` 增量解析优化：维护单调前进的 stable-prefix 边界——最后一个非空 markdown block 之前的内容定为稳定（memoized 的 `<Markdown>`，不再重新 lex），只有正在增长的最后一个 block 每个 delta 重新 `marked.lexer`。O(未稳定长度) 而非 O(全文)
- 无 markdown 语法的纯文本走快速路径跳过 lexer（`MD_SYNTAX_RE` 只扫前 500 字符）；token 结果按内容 hash LRU 缓存 500 条

### 2.3 thinking 内容展示（`AssistantThinkingMessage.tsx`）

- 默认（非 verbose、非 transcript）**折叠为一行**：`∴ Thinking` + `(ctrl+o to expand)`，**dim + italic**（`∴` = U+2234）
- verbose/transcript 展开：标题 `∴ Thinking…`（dim italic）+ 正文 `paddingLeft={2}` 的 `<Markdown dimColor>`（dim，非斜体正文）
- 流式 thinking：完整 thinking block 到达时经 `setStreamingThinking` 显示在动态区（Messages.tsx:714-719，以展开态渲染），流结束后**保留 30 秒**自动隐藏（REPL.tsx:852-864）；transcript 模式 `hidePastThinking` 只显示最后一个 thinking block，跨越用户 turn 即不显示旧 thinking

## 3. 工具运行中的进度

### 3.1 Bash 实时输出预览

- `BashModeProgress.tsx`（`!` bash 模式）：上方回显 `<bash-input>` 命令（`UserBashInputMessage`），下方 `ShellProgressMessage`；无进度数据时回退 BashTool 默认 `Running…`（dim，`BashTool/UI.tsx:147-149`）
- `ShellProgressMessage.tsx`（也用于 Bash 工具运行中 tick）：
  - 输出 `stripAnsi` 后**取最后 5 行**（verbose 显示全部），dim 色，`<Box height={5} overflow="hidden">`
  - 尾行状态：`+{N} lines`（被截断行数）或 `~{N} lines`；+ `ShellTimeDisplay`（运行 `formatDuration(elapsed)`，或 `(timeout 2m)`）+ 可选 `formatFileSize(totalBytes)`，`gap={1}` 横排
  - 无输出时：`Running… {elapsed}`
  - 包裹在 `OffscreenFreeze`（离屏冻结）+ `MessageResponse`（`⎿ ` 缩进）中
- **tick 频率**：Bash/Sleep 每秒发一条 progress 消息；REPL 对 ephemeral progress **同 tool 同 type 就地替换而非追加**（防止 messages 数组膨胀，REPL.tsx:2608-2627）

### 3.2 Subagent 进度行（`AgentProgressLine.tsx`）

两行结构，`paddingLeft={3}`：

```
   ├─ general-purpose (探索代码库) · 3 tool uses · 2.5k tokens
   │  ⎿  Grep(pattern: "foo")
```

- 树形符号：非末项 `├─`，末项 `└─`；第二行前缀：非末项 `│  ⎿  `，末项 `   ⎿  `（`⎿` = U+23BF）
- 第一行：agentType **bold**（有颜色时反色背景块），描述括号内；未 resolve 整行 dim；` · N tool uses · X tokens`
- 第二行状态：运行中显示 `lastToolInfo`（最近一次工具调用摘要）或 **`Initializing…`**；后台化 async agent 显示任务描述或 `Running in the background`；完成显示 `Done`
- Swarm/teammates 另有 `TeammateSpinnerTree/TeammateSpinnerLine`（每个 teammate 独立 spinner 行 + token 数，esc 可切换查看）

## 4. 中断呈现

- 常量（`utils/messages.ts:207-215`）：
  - `INTERRUPT_MESSAGE = '[Request interrupted by user]'`
  - `INTERRUPT_MESSAGE_FOR_TOOL_USE = '[Request interrupted by user for tool use]'`
  - esc 中断 → `createUserInterruptionMessage()` 追加一条 user 消息，内容为上述哨兵文本
- **渲染**（`InterruptedByUser.tsx`）：文本等于哨兵串的 user 消息不显示原文，渲染为一行 dim：**`Interrupted · What should Claude do instead?`**，包在 `MessageResponse height={1}`（`⎿ ` 缩进）里。`UserTextMessage.tsx:83`、`AssistantTextMessage.tsx:189`、`UserToolCanceledMessage.tsx` 三处复用

- 中断时已流出的部分文本固化为正式 assistant 消息（见 2.1）

## 5. 消息"定型"（动态区 → 静态区）

应用层的"静态"**不是** Ink `<Static>` 组件，而是 **React.memo 级别的永久跳过重绘**（终端 scrollback 提交在自研 ink 渲染器内部，见 05 篇）：

- `Messages.tsx:779 shouldRenderStatically(message, ...)` 判定规则：
  - transcript 模式一律 static
  - user/assistant/attachment：无关联 tool_use → static；tool_use **正在流式**（streamingToolUseIDs）或**执行中**（inProgressToolUseIDs）→ 动态；有未 resolve 的 PostToolUse hook → 动态；否则要求**所有 sibling tool_use 都已 resolve** 才 static（一个 assistant 消息里并行多个工具时整组一起定型）
  - `system`：`api_error` 永远动态（会被后续消息隐藏），其他 static
  - `grouped_tool_use`：全部 resolve 才 static
  - `collapsed_read_search`：prompt 模式下永不 static（防 turn 间闪烁）
- `MessageRow.tsx:155` 每行计算 `isStatic` 传给 `Message`；`Message.tsx:623` 自定义比较器：`if (prev.isStatic && next.isStatic) return true`——**两次渲染都 static 就直接跳过**（columns 变化、verbose 切换、screen 切换仍强制重绘）
- 即：一条消息在"其 tool_use 全部有结果、无 pending hook"那一刻定型，之后除 resize 外不再参与渲染

## 6. 队列消息展示（加载中排队的输入）

`PromptInput/PromptInputQueuedCommands.tsx`：

- 数据来自 `useCommandQueue()`；渲染位置在输入框区域上方，`<Box marginTop={1} flexDirection="column">`
- 每条排队命令用 `createUserMessage` 造成**合成 user 消息**，套 `QueuedMessageProvider` 用普通 `<Message isStatic={true} shouldShowDot={false}>` 渲染——外观与正式用户消息一致；`useMemo` 缓存防止流式重渲染导致 UUID 变化闪烁
- bash 模式排队项包成 `<bash-input>{cmd}</bash-input>` 显示
- 过滤：`isQueuedCommandVisible`——idle_notification 完全隐藏；task-notification **最多显示 3 行**（`MAX_VISIBLE_NOTIFICATIONS = 3`），超出折叠为合成消息 `+N more tasks completed`；isMeta 系统消息隐藏（channel 消息例外）
- 查看 teammate 转录时整块隐藏
- 输入框 placeholder 提示：**`Press up to edit queued messages`**（`usePromptInputPlaceholder.ts:54`，仅在有可编辑排队项且提示次数未超限时）
