## 1. 消息行通用格式

### 1.1 符号常量（`src/constants/figures.ts`）

| 常量 | 字符 | 用途 |
|---|---|---|
| `BLACK_CIRCLE` | macOS 用 `⏺`（U+23FA），Windows/Linux 用 `●` | 消息/工具行左侧圆点（⏺ 垂直居中更好但 Win/Linux 常不支持） |
| `BULLET_OPERATOR` | `∙` | 列表点 |
| `TEARDROP_ASTERISK` | `✻` | 系统提示行（如 compact 提示） |
| `REFERENCE_MARK` | `※` | away-summary 回顾标记 |
| `BLOCKQUOTE_BAR` | `▎`（U+258E） | Markdown 引用块左侧竖条 |
| 结果缩进符 | `⎿`（U+23BF） | 工具结果行前缀 |
| thinking 标记 | `∴`（U+2234） | "∴ Thinking" |

### 1.2 ⏺ 圆点（assistant 文本 / 工具行）

- **assistant 文本消息**（`src/components/messages/AssistantTextMessage.tsx:232`）：`<NoSelect fromLeftEdge minWidth={2}><Text color={isSelected ? 'suggestion' : 'text'}>⏺</Text></NoSelect>`——**⏺ 占 minWidth=2 的盒子**（⏺ + 1 空格），后接 Markdown 内容列。整行 `marginTop={addMargin?1:0}`（消息组之间空 1 行）。
- **工具调用行**的圆点由 `src/components/ToolUseLoader.tsx` 控制，**颜色即状态**：
  - 运行中（unresolved）：`color=undefined` + `dimColor`，且随 `useBlink` 闪烁（⏺ ↔ 空格交替）
  - 成功：`color="success"`（dark 主题 `rgb(78,186,101)`）
  - 失败：`color="error"`（dark `rgb(255,107,128)`）
  - 排队中（queued）：静态 dim ⏺（AssistantToolUseMessage.tsx:186）
  - 源码警告：`<dim>x</dim>` 紧跟 `<bold>y</bold>` 会导致 y 被 dim（chalk `\x1b[22m` 复用问题），实现时要避免。

### 1.3 ⎿ 结果行（`src/components/MessageResponse.tsx`）

所有工具结果/子行统一包在 `MessageResponse` 中：

```
<Text dimColor>{'  '}⎿ &nbsp;</Text>   ← 2 空格 + ⎿ + 空格 + nbsp，共 5 列，dim 色
```

- 前缀总宽 **5 字符**（`utils/terminal.ts` 注释确认："  ⎿ " = 5 chars）；前缀在 `NoSelect fromLeftEdge` 中（复制文本时不带 ⎿）。
- 通过 React Context 防止**嵌套 ⎿**：MessageResponse 后代再渲染 MessageResponse 时直接返回 children。
- 内容区 `flexShrink=1 flexGrow=1`，换行续行自动缩进对齐到第 6 列。

### 1.4 user 消息前缀（`HighlightedThinkingText.tsx` + `UserPromptMessage.tsx`）

- 前缀是 **`figures.pointer` + 1 空格**（Unix `❯`、旧 Windows 回退 `>`），颜色 `subtle`（dark `rgb(80,80,80)` 深灰；被 message-actions 选中时变 `suggestion` 蓝）。
- 整条消息有背景色 `userMessageBackground`（dark `rgb(55,55,55)`），`paddingRight={1}`，正文 `color="text"`。
- 超长输入截断：>10,000 字符时显示头 2500 + `… +N lines …` + 尾 2500（UserPromptMessage.tsx:64-70）。
- ultrathink 触发词逐字符彩虹色（`getRainbowColor`，rainbow_red…rainbow_violet 7 色循环）。
- **bash 模式输入**（`UserBashInputMessage.tsx`）：前缀 `<Text color="bashBorder">! </Text>`（dark 粉 `rgb(253,93,177)`），背景 `bashMessageBackgroundColor`（dark `rgb(65,60,65)`）。

### 1.5 错误消息格式

- API/系统错误：`MessageResponse`（⎿ 缩进）+ `<Text color="error">`（AssistantTextMessage.tsx 各 case）。多为 `height={1}` 单行，如 `Context limit reached · /compact or /clear to continue`。
- 通用 API 错误非 verbose 时截到 **1000 字符**（`MAX_API_ERROR_CHARS`）加 `…`，下方 `<CtrlOToExpand/>`。
- **工具错误兜底**（`FallbackToolUseErrorMessage.tsx`）：⎿ 缩进 + `color="error"`，剥掉 `<tool_use_error>`/`<error>` 标签，无 "Error: " 前缀则补上；非 verbose 只显示前 **10 行**（`MAX_RENDERED_LINES=10`），余下 dim 的 `… +N lines (ctrl+o to see all)`（ctrl+o 加粗）。
- **中断**（`InterruptedByUser.tsx`）：dim 的 `Interrupted · What should Claude do instead?`；取消/拒绝的工具（`UserToolCanceledMessage`/`FallbackToolUseRejectedMessage`）= ⎿ 行内放该文案。
- **thinking**（`AssistantThinkingMessage.tsx`）：折叠态 dim+italic `∴ Thinking (ctrl+o to expand)`；transcript/verbose 展开为 `∴ Thinking…` + `paddingLeft={2}` 的 dim Markdown 正文。

### 1.6 关键颜色 token（`src/utils/theme.ts` darkTheme :440-515）

- `claude: rgb(215,119,87)`（品牌橙）
- `permission/suggestion: rgb(177,185,249)`（内联代码/选中蓝紫）
- `success: rgb(78,186,101)`、`error: rgb(255,107,128)`、`warning: rgb(255,193,7)`
- `text: 白`、`inactive: rgb(153,153,153)`、`subtle: rgb(80,80,80)`
- `autoAccept: rgb(175,135,255)`、`planMode: rgb(72,150,140)`
- diff 四色见 §3
- 共 6 套主题：dark / light / 各自 daltonized / 各自 ansi

## 2. 工具调用展示

### 2.1 标题行结构（`AssistantToolUseMessage.tsx:182-244`）

```
[⏺(loader, minWidth 2)][bold 工具名]([参数摘要])[可选 tag]
```

- 工具名 `bold`、`wrap="truncate-end"`；参数摘要在字面括号里：`<Text>({renderedToolUseMessage})</Text>`；摘要为 `''` 时不显示括号。
- `userFacingName() === ''` 的工具（如 TodoWrite）整行隐藏。
- 未 resolved 时下方追加状态行（⎿ + dim）：`Waiting for permission…` / `Bash classifier checking…` / 工具自定义 progress；queued 时 `Waiting…`。

### 2.2 各工具参数摘要

- **Bash**（`src/tools/BashTool/UI.tsx`）：直接显示命令文本；非 verbose 截断为 **最多 2 行 / 160 字符**（`MAX_COMMAND_DISPLAY_LINES=2, MAX_COMMAND_DISPLAY_CHARS=160`）+ `…`；sed 就地编辑命令显示为文件路径。运行中默认 `⎿ Running…`（dim），可后台化时提示 `(ctrl+b to run in background)`（paddingLeft 5）。
- **Read**（`src/tools/FileReadTool/UI.tsx`）：`Read(相对路径)`，路径是 `FilePathLink`（OSC8 超链接）；verbose 加 `· lines 5-25`；PDF 加 `· pages 1-5`。userFacingName 动态：普通 `Read`、计划文件 `Reading Plan`、agent 输出 `Read agent output`。
- **Grep**：`pattern: "xxx", path: "yyy"`（非 verbose 路径 `getDisplayPath` 相对化）。
- **Glob**（userFacingName 是 `Search`）：`pattern: "xxx", path: "yyy"`。
- **Edit/Write**（`src/tools/FileEditTool/UI.tsx`）：`Update(路径)` / `Create(路径)`，FilePathLink。

### 2.3 结果摘要行

- **Read**：⎿ `Read **N** lines`（N 加粗）/ `Read image (34KB)` / `Read **N** cells` / dim `Unchanged since last read`。
- **Grep/Glob**（`GrepTool/UI.tsx` SearchResultSummary）：⎿ `Found **N** files`（count 加粗、自动单复数），count 模式 `Found **N** matches across **M** files`；非 verbose 且 N>0 时行尾跟 `(ctrl+o to expand)`；verbose 才在下面 `marginLeft={5}` 列出完整内容。
- **Bash**（`BashToolResultMessage.tsx` + `shell/OutputLine.tsx`）：stdout 一块、stderr 一块（`color="error"`），各自 ⎿ 缩进；空输出单行 dim `(No output)` 或 `Done` 或 `Running in the background (↓ to manage)`；stdout 会尝试 JSON pretty-print（≤10K 字符）、URL 转超链接、剥 underline ANSI。

### 2.4 输出截断规则（`src/utils/terminal.ts`）

- `MAX_LINES_TO_SHOW = 3`：非 verbose 的 bash/工具输出只显示**按终端宽度折行后（wrapWidth = columns - 10）的前 3 行**
- 剩 1 行时直接显示第 4 行（避免 "+1 line" 的尴尬）
- 否则追加 chalk.dim 的 `… +N lines (ctrl+o to expand)`（N 为折行后剩余行数；快捷键文本从 keybinding 系统取）
- 性能上限：只处理 `3 × wrapWidth × 4` 字符，超出部分按估算行数计入 N

### 2.5 Todo 列表渲染

当前版本 **TodoWrite 在消息流中不渲染任何内容**（`src/tools/TodoWriteTool/TodoWriteTool.ts`：`userFacingName() => ''`、`renderToolUseMessage() => null`）。todo 列表显示在 prompt 上方的任务面板 `src/components/TaskListV2.tsx`：

- 图标（:220-241 `getTaskIcon`）：completed = `figures.tick`（✔）色 `success`；in_progress = `figures.squareSmallFilled`（◼）色 `claude` 橙；pending = `figures.squareSmall`（◻）无色。
- 文本样式（:313）：`bold={isInProgress}`、`strikethrough={isCompleted}`、`dimColor={isCompleted || isBlocked}`；被 block 时后缀 dim `› blocked by …`；标题栏显示 `N completed / M pending` 计数（bold 数字）。
- 更多细节（截断策略、owner 显示）见 06 篇 §5。

## 3. 文件编辑 Diff

### 3.1 架构

- 主路径：`StructuredDiff.tsx` 调 Rust NAPI 模块 `color-diff-napi`（`ColorDiff`/`ColorFile`，带语法高亮的 diff 渲染，输出 ANSI 行数组，按主题名取 `getSyntaxTheme`）；可被 `CLAUDE_CODE_SYNTAX_HIGHLIGHT=false` 或 settings `syntaxHighlightingDisabled` 关掉。
- 回退路径：`src/components/StructuredDiff/Fallback.tsx`（纯 TS，格式与 Rust 版一致），**格式规范以此为准**。

### 3.2 行格式（Fallback.tsx:389-419）

每行 = `[行号右对齐 padStart(maxWidth)][1 空格][符号][代码][补齐空格到全宽]`：

- gutter 宽 = 最大行号位数 + 3（marker 1 + 两侧空格，StructuredDiff.tsx:46-49）
- 符号：add `+`、remove `-`、context 一个空格
- **背景色刷满整行**（含右侧 padding）：add → `diffAdded`（dark 深绿 `rgb(34,92,43)`；light 浅绿 `rgb(105,219,124)`），remove → `diffRemoved`（dark 深红 `rgb(122,41,54)`）；dim 模式（如被拒绝的 diff）用 `diffAddedDimmed`/`diffRemovedDimmed`；context 行无背景、gutter dim
- 行号编号：remove 块与后续 add 块共享同段行号（`numberDiffLines`）
- gutter 包 `NoSelect`（全屏选择文本时不带行号）

### 3.3 Word-level diff（:237-347）

- 相邻的 remove/add 行两两配对，`diffWordsWithSpace` 做词级 diff
- 变化比例 > **0.4**（`CHANGE_THRESHOLD`）则放弃词级、整行高亮
- 变化词叠加更亮的背景：`diffAddedWord`（dark `rgb(56,166,96)`）/ `diffRemovedWord`（dark `rgb(179,89,107)`），未变部分只有整行底色

### 3.4 上下文与外框

- 上下文行数 `CONTEXT_LINES = 3`（`src/utils/diff.ts:9`；单 hunk 时 context 放宽到 100_000 即全显）
- Edit 权限对话框中 diff 用上下虚线框：`borderStyle="dashed" borderColor="subtle"`，只有上下边（`FileEditToolDiff.tsx` DiffFrame）
- 编辑完成消息（`FileEditToolUpdatedMessage.tsx`）：⎿ 内首行 `Added **N** lines, removed **M** lines`（数字加粗），下接 diff，diff 宽度 = `columns - 12`

## 4. Markdown 终端渲染

### 4.1 formatToken（`src/utils/markdown.ts`）——除表格外都编译成 ANSI 字符串

| 元素 | 呈现 |
|---|---|
| H1 | `chalk.bold.italic.underline` + 两个换行 |
| H2 / H3+ | `chalk.bold` + 两个换行（# 号不显示） |
| **strong** | `chalk.bold` |
| *em* | `chalk.italic` |
| `codespan` | 前景色 `permission`（dark 蓝紫 `rgb(177,185,249)`），无背景 |
| 代码块 | cli-highlight（highlight.js）按语言高亮；不支持的语言按 plaintext；无高亮器时原文 |
| blockquote | 每行前缀 dim `▎` + 空格，正文 `chalk.italic`（不 dim——dim 在暗主题几乎看不见） |
| hr | 字面 `---` |
| 链接 | 有显示文本→OSC 8 超链接包住文本；文本=URL→直接 URL 超链接；mailto 剥成纯邮箱文本 |
| 无序列表 | `-` + 空格，每层缩进 **2 空格** |
| 有序列表 | 深度 0/1 用数字、深度 2 用字母（a.）、深度 3 用罗马数字（i.），后跟 `.` |
| 删除线 `~~` | **禁用**（模型常用 ~ 表约数） |
| `owner/repo#123` | 自动转 GitHub issue 超链接 |
| 块间距 | Markdown.tsx 外层 `Box gap={1}`，块之间空一行 |

### 4.2 表格（`src/components/MarkdownTable.tsx`，作为 React 组件单独渲染）

- 盒线字符全边框：`┌─┬┐ / │ / ├─┼┤ / └─┴┘`，每行数据间也画 `├┼┤` 分隔线
- 单元格两侧各 1 空格 padding；表头**居中**，数据按 md 对齐标记（默认左）
- 列宽算法：ideal（内容全宽）→ 不够则 min（最长单词）+ 按溢出比例分配 → 再不够按比例硬折行；`SAFETY_MARGIN=4`、`MIN_COLUMN_WIDTH=3`、单元格折行超 `MAX_ROW_LINES=4` 行时切**垂直格式**（`label:` 加粗 + 值，行间 `─`×min(width-1,40) 分隔）
- 多行 cell 垂直居中；整表拼成字符串用一个 `<Ansi>` 输出防止 Ink 折行撕裂

### 4.3 语法高亮

- Markdown 代码块：`cli-highlight`（highlight.js 默认配色，随终端 ANSI）；异步加载，Suspense fallback 先渲染无高亮版。
- 文件预览/diff：Rust `color-diff-napi` 的 `ColorFile`/`ColorDiff`，主题按 dark/light 系列名取（`HighlightedCode.tsx`，默认宽 80，回退 `HighlightedCode/Fallback.tsx`）。

## 5. 折叠与展开

### 5.1 CtrlOToExpand（`src/components/CtrlOToExpand.tsx`）

- 组件版：dim 的 `(ctrl+o to expand)`（快捷键文本动态取自 keybinding `app:toggleTranscript`）；在 SubAgent 内部或虚拟列表中**不渲染**（防提示泛滥）。
- 字符串版 `ctrlOToExpand()`：`chalk.dim('(ctrl+o to expand)')`，供 renderTruncatedContent 拼接。
- `KeyboardShortcutHint` 通式：`(shortcut to action)` 或 `shortcut to action`，可 bold shortcut。

### 5.2 Transcript 模式（ctrl+o，`src/components/Messages.tsx`）

- `screen === 'transcript'`；进入后显示完整历史：thinking 全文（只有**最后一个** thinking 块显示，历史 thinking 隐藏 `hideInTranscript`）、被折叠的 read/search 组逐条展开、完整错误文本
- transcript 内还有 Ctrl+E `transcript:toggleShowAll` 二级开关（截断 cap）；有选中消息时"expanded 覆盖 verbose"
- **verbose 模式**差异（`--verbose` / 配置）：不进 transcript 也直接显示全量——OutputLine 不截 3 行、Bash 命令不截 160 字符、路径显示绝对路径、Grep 结果全列出（marginLeft 5）、API 错误不截 1000 字符

### 5.3 Read/Search 折叠组（`src/utils/collapseReadSearch.ts` + `messages/CollapsedReadSearchContent.tsx`）

连续的 Read/Grep/Glob/List/REPL/MCP/Bash（全屏模式）调用合并为一行摘要：

- 进行中：闪烁 loader + 正常色 `Searching for **3** patterns, reading **2** files…` + `(ctrl+o to expand)`，下方 ⎿ 行（5 列 gutter）dim 显示当前命令 hint（cap `MAX_HINT_CHARS=300` ≈5 行），bash 耗时 ≥2s 追加 ` (3s · 42 lines)`
- 完成后：无 loader（minWidth 2 占位），整行 dim，动词变过去式 `Searched for …, read …`；数字都 bold
- git 操作打头（`committed <sha>`、`pushed to <branch>`、`merged PR #N`…），内存操作殿后（`recalled N memories`）
- hook 统计另起 ⎿ 行：`Ran N PreToolUse hooks (1.2s)`

## 6. 图片展示（`src/components/ClickableImageRef.tsx`）

- 粘贴的图片在消息里渲染为字面 `[Image #N]`
- 终端支持 OSC 8 且图片已存盘（`getStoredImagePath`）→ 包成 `file://` 超链接（点击用系统查看器打开），fallback 普通文本
- 选中态 `inverse` + `bold`；可传 backgroundColor（在 user 消息背景内使用）
- Bash 输出探测到图片数据时显示 ⎿ dim `[Image data detected and sent to Claude]`

## 7. 实现要点速查（给 Transup）

1. 左列 gutter 恒为 2 列：`⏺ `；结果行恒为 5 列 `  ⎿  `（dim），续行缩进对齐。
2. 状态用**圆点颜色**表达：运行=dim 闪烁、成功=绿、错误=红、排队=dim 静止；不要额外文字。
3. 工具行格式：`⏺ ToolName(args)`，ToolName bold，args 不 bold；结果单行摘要中的**数字 bold**。
4. 截断统一话术：`… +N lines (ctrl+o to expand)`，dim；输出默认 3 行，错误默认 10 行，Bash 命令 2 行/160 字符。
5. diff：`行号 + 符号 + 代码`，整行背景色（深色主题用暗红/暗绿，变化词用亮一档的红/绿背景），上下文 3 行，无行内 │ 分隔。
6. user 消息：灰底 + `❯ ` 前缀（subtle 色）；bash 输入：`! ` 粉色前缀 + 微紫灰底。
7. Markdown：标题只用 bold（H1 加 italic+underline），inline code 用蓝紫前景色，引用块 `▎` + italic，列表 2 空格/层缩进，表格全盒线框。
