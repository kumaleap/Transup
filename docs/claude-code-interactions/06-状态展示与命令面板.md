## 1. 状态展示

### 1.1 自定义 statusline 机制

关键文件：
- `src/components/StatusLine.tsx`（渲染组件）
- `src/utils/hooks.ts:4584` `executeStatusLineCommand()`（命令执行）
- `src/commands/statusline.tsx`（/statusline 命令，实际是转发给 `statusline-setup` 子 agent 来帮用户写配置）

行为：
- 仅当 `settings.statusLine` 存在时显示（`statusLineShouldDisplay()`）；assistant/KAIROS 模式下隐藏。
- 每次触发时构造一个 JSON 输入（`buildStatusLineCommandInput`），字段极其丰富：`model{id,display_name}`、`workspace{current_dir,project_dir,added_dirs}`、`version`、`output_style`、`cost{total_cost_usd,total_duration_ms,total_api_duration_ms,total_lines_added,total_lines_removed}`、`context_window{total_input_tokens,total_output_tokens,context_window_size,current_usage,used_percentage,remaining_percentage}`、`exceeds_200k_tokens`、`rate_limits{five_hour,seven_day}`（各含 used_percentage/resets_at）、`vim{mode}`、`agent{name}`、`remote{session_id}`、`worktree{...}`、`session_name`。
- 该 JSON 通过 stdin 传给用户配置的 shell 命令（`type: 'command'`），**超时 5 秒**；exit code 0 时取 stdout，trim 后按行拼接；非 0 静默丢弃。
- 更新触发条件：最后一条 assistant 消息 id 变化 / permission mode 变化 / vim mode 变化 / 主循环模型变化，**300ms debounce**，且用 AbortController 取消 in-flight 执行。
- 渲染：`<Text dimColor wrap="truncate"><Ansi>{text}</Ansi></Text>`——支持命令输出中的 ANSI 颜色；`padding` 可由 `settings.statusLine.padding` 配置（paddingX，默认 0）。全屏模式下加载中会保留一行空白防止布局跳动。
- 安全：所有 hook（含 statusline）要求 workspace trust；未信任时显示 low priority 通知 `statusline skipped · restart to fix`。`disableAllHooks` 时仅 managed（policySettings）的 statusline 可运行。
- 位置：`PromptInputFooter`（输入框下方 footer），用 `memo` 隔离重渲染。

### 1.2 上下文用量展示（/context）

关键文件：`src/commands/context/context.tsx`（入口）、`src/components/ContextVisualization.tsx`（渲染）、`src/utils/analyzeContext.ts`（分析 + 网格构建）

行为：
- /context 先把消息转换为「API 实际所见视图」：取 compact 边界后的消息 → contextCollapse 的 projectView → microcompact，避免虚高。然后 `analyzeContextUsage()` 分析，`renderToAnsiString()` 渲染为静态 ANSI 字符串输出到 transcript（不是常驻面板）。
- 布局（自上而下）：
  1. 标题 `Context Usage`
  2. **方块网格**：宽度取决于终端（`GRID_WIDTH`），高度 10 行（窄终端 5 行、1M 上下文固定 10 行）；每格字符：空闲 `⛶`、Autocompact buffer 保留区 `⛝`、已用类别按填充度 ≥0.7 用 `⛁` 否则 `⛀`，并按类别着色

  5. 追加分区：`MCP tools · /mcp`（区分 Loaded / Available on-demand）、`Custom agents · /agents`、`Memory files · /memory`、`Skills · /skills`，各按来源（Project/User/Managed/Plugin/Built-in）分组，条目形如 `└ name: 1.2k tokens`

## 4. 自动更新与退出

### 4.1 AutoUpdater
`src/components/AutoUpdater.tsx`（另有 `NativeAutoUpdater.tsx`、`PackageManagerAutoUpdater.tsx`、`AutoUpdaterWrapper.tsx` 按安装方式分流）：
- 启动时检查一次 + **每 30 分钟**轮询；比较 `MACRO.VERSION` 与 npm channel（`autoUpdatesChannel`，默认 latest）最新版；支持服务端 `maxVersion` 熔断上限、`shouldSkipVersion` 跳版本。
- 按检测到的安装类型选择更新方式：npm-local（`~/.claude/local`）/ npm-global / native（转 NativeAutoUpdater）/ development（跳过）。用 ref 防止并发更新。
- UI（一行）：更新中 dim `Auto-updating…`；成功绿色 `✓ Update installed · Restart to apply`；失败红色 `✗ Auto-update failed · Try claude doctor or npm i -g ...`（本地安装时提示 `cd ~/.claude/local && npm update ...`）；verbose 下附 `globalVersion: x · latestVersion: y`。

## 5. Todo 列表呈现（TaskListV2）

`src/components/TaskListV2.tsx`（TodoV2 启用时；V1 的 TodoWriteTool `userFacingName()` 返回空串、`renderToolUseMessage` 返回 null，即工具调用本身在 transcript 中不渲染标题行，全部 todo 全完成时自动清空列表）：

- **符号与样式**（`getTaskIcon`）：
  - completed：`figures.tick ✔`，`success` 绿色，主题文本 **删除线 + dim**
  - in_progress：`figures.squareSmallFilled ◼`，`claude` 品牌色，文本 **bold**
  - pending：`figures.squareSmall ◻`，无色
- **每行结构**：`icon subject (@owner) › blocked by #2, #3`；owner 仅在 ≥60 列且该 teammate 仍在运行时显示、按 teammate 颜色着色；被阻塞任务 dim 并列出未完成的 blocker id。in_progress 且未阻塞时下方缩进一行 dim 显示实时活动摘要 `  {activity}…`（来自 teammate 的 recentActivities 归并）。
- **截断策略**：可见条数 `maxDisplay = clamp(3, rows-14, 10)`（rows≤10 则 0）；超出时优先级 **最近 30 秒内完成的（保留展示 30s TTL，定时器精确到期重渲）→ in_progress → pending（未被阻塞的排前）→ 更早完成的**；隐藏部分汇总为 ` … +2 in progress, 3 pending, 5 completed`。
- **位置**：spinner 运行时嵌在 spinner 正下（缩进的 MessageResponse 内）；空闲 + ctrl+t 展开时以 standalone 形式出现在输入框上方，带 `{N} tasks (X done, Y in progress, Z open)` 头。
- Spinner 动词直接复用当前 in_progress todo 的 `activeForm`（如 "Refactoring auth module…"），下一条 pending 显示为 `Next: {subject}`——todo 系统与主活动行深度联动。

## 6. 对 Transup 的关键可借鉴点

1. **statusline 即 hook**：外部命令 + 结构化 JSON stdin + 5s 超时 + 300ms debounce + ANSI 透传 + trust 门禁，是低成本高扩展的用户自定义方案。
2. **键位全部经 context 化的 keybinding 注册表**（Global/Chat/Select/Confirmation/Settings/Tabs/Task/Footer…），UI 提示用 `useShortcutDisplay` 反查，用户重绑定自动反映到所有 hint。
3. **叶子层性能隔离**：50ms 动画时钟只在 SpinnerAnimationRow；StatusLine memo；shimmer 抽成独立组件避免整棵对话框 20fps 重渲。
4. **任务是统一抽象**：七种后端共用 `BackgroundTaskState` + 每类型一个摘要组件 + 每类型一个详情 Dialog；列表→详情的 skippedListOnMount 智能导航值得照抄。
5. **完成通知面向模型而非弹窗**：`<task_notification>` XML 注入下一轮，UI 只动 pill 计数。
6. **一切宽度自适应**：truncateToWidth/truncatePathMiddle/渐进裁剪状态项，是终端 UI 可读性的核心纪律。
